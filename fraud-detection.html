Project Summary

A complete fraud detection pipeline from raw transactions to anomaly extract.

The project starts from `data/bank_transactions_data.csv`, a dataset with 2,512 banking transaction records and fields such as transaction dates, amounts, merchant identifiers, device information, login attempts, account balance, and related transaction metadata.

The workflow enriches the raw dataset with engineered features, behavioral flags, z-score flags, Isolation Forest labels, and a final anomaly target used to extract suspicious transactions for review.

Workflow

Notebook-by-notebook pipeline

The repository is organized into five notebooks. Each notebook adds a specific layer to the pipeline and writes a downstream data artifact.

01

Data understanding & cleaning

Quality checks, duplicate removal, date conversion, label encoding, MinMax scaling, `time_diff`, and `location_change`.

Output: data/clean_transactions_data.csv
02

Exploratory data analysis

Distribution checks, outlier review, correlations, hourly activity, and behavioral suspicious flags.

Output: data/clean_transactions_with_flags.csv
03

Statistical methods

Z-score calculations on selected numeric fields and binary flags using the `abs(z) > 3` rule.

Output: data/transactions_with_zscores.csv
04

Isolation Forest

Unsupervised anomaly detection and combined final anomaly flag generation from rule-based and model-based signals.

Output: data/transactions_with_all_flags_final.csv.csv
05

Supervised anomaly model

Stratified train/test split, RandomForestClassifier, classification report, confusion matrix, and feature importance review.

Output: transactions_anomalies_only.csv

EDA

Exploratory analysis and rule-building context

The exploratory phase is used to understand distributions, identify outliers, inspect merchant behavior, and build practical fraud flags before model training.

Modeling

Isolation Forest, final flags, and supervised validation

The modeling stage combines statistical flags, behavioral rules, unsupervised anomaly detection, and supervised Random Forest classification to produce and validate the final anomaly extract.

Data Artifacts

Generated datasets across the pipeline.

  • `data/bank_transactions_data.csv`: raw dataset with 2,512 rows and 16 columns.
  • `data/clean_transactions_data.csv`: cleaned and engineered dataset with 2,512 rows and 18 columns.
  • `data/clean_transactions_with_flags.csv`: EDA flags added, 2,512 rows and 31 columns.
  • `data/transactions_with_zscores.csv`: z-score features and flags added, 2,512 rows and 47 columns.
  • `data/transactions_with_all_flags_final.csv.csv`: final combined flag dataset, 2,512 rows and 50 columns.
  • `transactions_anomalies_only.csv`: final suspicious transaction extract with 947 rows and 28 columns.
